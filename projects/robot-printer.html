02. Case Study: Hardware Automation

Direct-to-Printer Robot

Klien: PT. Mada Wikri Tunggal
Peran: Backend & Automation Engineer
Stack: PowerShell, SumatraPDF, Laravel REST API
Frek. Polling: 0.33 Hz (Interval 3 Detik)

Masalah Operasional Awal

Saat dokumen Surat Perintah Kerja (SPK) dirilis dari sistem ERP pusat (di-host di cloud), dokumen tersebut harus segera dicetak secara fisik di printer lantai produksi masing-masing departemen. Sebelum adanya otomatisasi ini:

  • Prosedur Berbelit: Operator stasiun kerja harus membuka browser web, login ke sistem ERP, mencari nomor SPK, mengunduh file PDF secara manual, membuka PDF Reader (seperti Adobe Reader), memanggil dialog box cetak (Ctrl+P), mencocokkan pengaturan printer lokal, dan mengklik konfirmasi cetak.
  • Friksi Operasional: Operator pabrik sering kali gagap mengoperasikan komputer, yang berujung pada kekeliruan cetak (salah ukuran kertas, orientasi terpotong, atau salah printer tujuan).
  • Bottleneck Penugasan: Waktu rilis SPK dari admin hingga operator memegang dokumen kerja fisik memakan waktu rata-rata 2 hingga 3 menit.

Pendekatan Arsitektur Teknologi yang Dipilih

Solusi yang dirancang adalah agen PowerShell lokal yang berjalan sebagai background service pada server lokal pabrik yang terhubung ke printer fisik. Agen ini melakukan komunikasi dua arah dengan RESTful API server ERP.

1. Laravel Cloud ERP
Admin merilis SPK. Job cetak dimasukkan ke tabel antrean TPrintQueue.
→
2. PS Script (3s Agent)
PowerShell lokal melakukan polling GET /api/print-queue/pending tiap 3 detik.
→
3. Silent Printer Agent
Mengunduh PDF, memanggil SumatraPDF CLI untuk cetak silent tanpa GUI dialog box.
→
4. Server Callback
Mengirim status sukses POST /api/print-queue/complete/{id} ke server.

Karena SumatraPDF dijalankan secara silent via CLI, operator tidak terganggu oleh kemunculan GUI Reader. Dokumen langsung keluar dari printer pabrik secara instan.

Tantangan Teknis & Rekayasa OS

Di balik kesederhanaan alur di atas, integrasi otomatisasi cetak berbasis web pada lingkungan server Windows sering kali menghadapi kendala teknis yang rumit.

1. Kendala Windows Session 0 Isolation (Headless Context):
Aplikasi pembaca PDF standar seperti Acrobat Reader atau SumatraPDF pada dasarnya tidak dirancang untuk berjalan dalam skenario tanpa kepala (headless). Ketika script PowerShell dipicu secara otomatis oleh sistem Windows Task Scheduler atau Windows Service, script tersebut berjalan pada Session 0. Windows secara ketat memblokir akses dari Session 0 ke subsistem Graphical Device Interface (GDI) atau spooler printer fisik. Hal ini mengakibatkan print job menggantung (freeze) tanpa pesan error.

Solusi Session 0 Isolation: Konfigurasi parameter interaksi layanan disesuaikan dengan mendaftarkan script melalui utilitas pembungkus service yang secara eksplisit mengaktifkan interaksi desktop (*Allow service to interact with desktop*) atau dialihkan melalui sesi desktop terotentikasi yang berjalan di latar belakang menggunakan Windows Auto-Logon terenkripsi.

2. Kendala Otentikasi Lapis Kedua (Double-Hop / Second-Hop Problem):
Ketika script PowerShell dipicu melalui koneksi jarak jauh (seperti OpenSSH / WinRM), kredensial user tidak dapat didelegasikan di luar hop pertama. Akibatnya, script kehilangan kredensial untuk mengakses printer jaringan (*Network Shared Printer*) dan menghasilkan error "Access Denied".

Solusi Second-Hop: Masalah delegasi kredensial diselesaikan dengan memodifikasi autentikasi CredSSP (Credential Security Support Provider) pada host lokal dan server target, atau menyusun objek kredensial aman PSCredential secara langsung dalam script untuk memaksa otentikasi spooler printer secara eksplisit: 

# Mengatasi Double-Hop: Deklarasi kredensial lokal eksplisit untuk Windows Spooler
$PasswordSec = ConvertTo-SecureString "P@sswordPabrik123" -AsPlainText -Force
$Creds = New-Object System.Management.Automation.PSCredential("Administrator", $PasswordSec)

# Eksekusi SumatraPDF silent print menggunakan kredensial aman
Start-Process -FilePath "$SumatraPath" -ArgumentList "-print-to `"$PrinterName`" -silent `"$safePdf`"" -Credential $Creds -NoNewWindow -Wait

Hasil Akhir yang Terukur

Pemrosesan antrean cetak berjalan secara otomatis dan stabil di latar belakang server Windows. Kapasitas dan efisiensi waktu pemrosesan sistem dianalisis secara kuantitatif:

[ FREKUENSI JAJAK PENDAPAT (POLLING FREQUENCY) ]
fpolling = 1 / tinterval = 1 / 3 s ≈ 0.33 Hz

[ FORMULA EFISIENSI DISTRIBUSI SPK ]
ΔTdistribusi = (Tmanual - Totomatis) / Tmanual × 100%
ΔTdistribusi = (150 s - 15 s) / 150 s × 100% = 90%

Dampak Terukur Hasil Penerapan:

  • Efisiensi Waktu Distribusi SPK: Durasi dari SPK dirilis hingga tercetak fisik berkurang sebesar 90% (dari rata-rata 2.5 menit menjadi <15 detik).
  • Reduksi Kesalahan Cetak: Kesalahan pengaturan orientasi kertas, printer, atau PDF rusak turun menjadi 0% karena semua parameter cetak sudah di-hardcode di sisi script server.
  • Pencegahan Kemacetan Lini Produksi: Menghilangkan bottleneck cetak SPK harian, memastikan operator langsung mendapatkan lembar kerja secara instan tanpa perlu melangkah ke komputer admin.
