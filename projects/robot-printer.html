02. Case Study: Hardware Automation

Direct-to-Printer Robot

Klien: PT. Mada Wikri Tunggal
Peran: Backend & Automation Dev
Stack: PowerShell, SumatraPDF, Laravel REST API
Jeda Polling: 3 Detik (Real-time Agent)

The Problem (Masalah Bisnis)

Saat dokumen Surat Perintah Kerja (SPK) dirilis dari sistem ERP pusat (yang di-host di cloud), dokumen tersebut harus dicetak secara fisik di printer lantai produksi masing-masing departemen. Sebelum otomatisasi ini dibuat:

  • Prosedur Berbelit: Operator pabrik harus secara manual menyegarkan (refresh) halaman ERP, mengunduh file PDF SPK, membuka aplikasi PDF Reader, memanggil dialog pencetakan (Ctrl+P), mencocokkan printer departemen, dan menekan cetak.
  • Friksi Digital: Operator produksi sering kali kurang terbiasa mengoperasikan antarmuka komputer yang rumit, sehingga sering terjadi kesalahan cetak (salah ukuran kertas, orientasi landscape terpotong, atau salah printer).
  • Bottleneck Waktu: Waktu yang terbuang sejak admin merilis dokumen hingga operator memegang lembar fisik SPK rata-rata memakan waktu 2 hingga 3 menit.

The Architecture (Otomasi Cetak Tanpa Pop-up)

Solusi yang dirancang adalah agen PowerShell lokal yang berjalan sebagai background service pada server lokal pabrik yang terhubung ke printer fisik. Agen ini melakukan komunikasi dua arah dengan RESTful API server ERP.

1. Laravel Cloud ERP
Admin merilis SPK. Job cetak dimasukkan ke tabel antrean TPrintQueue.
→
2. PS Script (3s Agent)
PowerShell lokal melakukan polling GET /api/print-queue/pending tiap 3 detik.
→
3. Silent Printer Agent
Mengunduh PDF, memanggil SumatraPDF CLI untuk cetak silent tanpa GUI dialog box.
→
4. Server Callback
Mengirim status sukses POST /api/print-queue/complete/{id} ke server.

Karena SumatraPDF dijalankan secara silent via CLI, operator tidak terganggu oleh kemunculan GUI Reader. Dokumen langsung keluar dari printer pabrik secara instan (<10 detik sejak dirilis oleh admin di cloud).

The Code (Agen Polling & Callback API)

Berikut adalah bagian inti dari script PowerShell yang berjalan secara terus-menerus di server lokal pabrik untuk memproses pekerjaan cetak: 

# RobotPrinter.ps1
$ServerUrl = "https://s2smfg.biz.id"
$PrinterName = "EPSON L3250 Series (Copy 1)"
$SumatraPath = "C:\Program Files\SumatraPDF\SumatraPDF.exe"
$SleepSeconds = 3

while ($true) {
    try {
        # Mengambil daftar antrean SPK yang pending
        $response = Invoke-RestMethod -Uri "$ServerUrl/api/print-queue/pending" -Method Get
        if ($response.status -eq 'success' -and $response.data.count -gt 0) {
            foreach ($spkId in $response.data) {
                $pdfUrl = "$ServerUrl/api/print-queue/download/$spkId"
                $tempFile = "$PSScriptRoot\spk_print_$spkId.pdf"

                # Unduh file PDF secara terisolasi
                Invoke-WebRequest -Uri $pdfUrl -OutFile $tempFile -UseBasicParsing
                
                # Kirim ke printer lokal pabrik menggunakan SumatraPDF CLI secara silent
                $wsh = New-Object -ComObject WScript.Shell
                $wsh.Run("`"$SumatraPath`" -print-to `"$PrinterName`" -silent `"$tempFile`"", 0, $true)
                
                # Jeda agar spooler Windows selesai memproses job
                Start-Sleep -Seconds 5

                # Konfirmasi status cetak selesai ke cloud ERP
                Invoke-RestMethod -Uri "$ServerUrl/api/print-queue/complete/$spkId" -Method Post
                Remove-Item $tempFile -Force
            }
        }
    }
    catch {
        # Server offline or connection lost - ignore to avoid crash
    }
    Start-Sleep -Seconds $SleepSeconds
}

Berikut adalah implementasi endpoint REST API pada controller backend Laravel yang mengelola antrean cetak: 

// App/Http/Controllers/Api/PrintQueueController.php
namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Models\TPrintQueue;

class PrintQueueController extends Controller
{
    /**
     * Mengambil daftar ID pekerjaan cetak yang masih tertunda (pending).
     */
    public function getPending()
    {
        $pendingJobs = TPrintQueue::where('status', 'pending')
            ->orderBy('created_at', 'asc')
            ->pluck('spk_id');
            
        return response()->json([
            'status' => 'success',
            'data' => $pendingJobs
        ]);
    }

    /**
     * Memperbarui status pekerjaan cetak menjadi selesai (printed).
     */
    public function markComplete($spk_id)
    {
        TPrintQueue::where('spk_id', $spk_id)
            ->where('status', 'pending')
            ->update(['status' => 'printed', 'updated_at' => now()]);
            
        return response()->json([
            'status' => 'success',
            'message' => 'Pekerjaan cetak ditandai sukses'
        ]);
    }
}

Mengapa otomatisasi ini sukses?

  • Zero Interface Operator: Operator tidak perlu membuka browser atau menekan satu tombol pun. Printer akan mencetak lembar kerja fisik secara otomatis dalam hitungan detik setelah admin memproses SPK.
  • Konsumsi Memori Rendah: Menggunakan SumatraPDF CLI yang jauh lebih ringan daripada Adobe Acrobat Reader, mencegah server lokal kelebihan beban RAM.
  • Resiliensi Koneksi: PowerShell menggunakan blok try/catch kosong pada level luar untuk mencegah script terhenti (crash) saat terjadi putus jaringan internet sementara di pabrik.
