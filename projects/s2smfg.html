01. Case Study: Enterprise System

S2SMFG (Procure to Ship ERP)

Klien: PT. Mada Wikri Tunggal
Peran: Lead Full-Stack Developer
Stack: Laravel, MySQL/MariaDB, AJAX
Skala: 78 Tabel | 50+ Daily Users

The Problem (Masalah Bisnis)

Sebelum sistem S2SMFG dideploy, seluruh alur kerja pelacakan material dan produksi di pabrik PT. Mada Wikri Tunggal masih berbasis kertas manual. Lembar "Kartu Travel" digunakan untuk mencatat perpindahan barang jadi dari mesin cetak ke area karantina QC, lalu ke gudang barang jadi.

Metode manual ini menyebabkan beberapa masalah kritis pada rantai pasok:

  • Kehilangan Data Fisik: Kartu travel kertas sering kotor, robek, atau bahkan hilang di area lantai pabrik yang sibuk, sehingga melumpuhkan pelacakan riwayat lot produksi.
  • Delay Sinkronisasi Inventaris: Stok barang di gudang hanya direkap sekali dalam 24 jam ke dalam spreadsheet Excel. Manajemen tidak memiliki data riil saat melakukan perencanaan produksi (PPIC).
  • Human Error Tinggi: Kesalahan pengetikan kode part yang mirip (misalnya part dengan suffix "-A" tertukar dengan "-B") menyebabkan selisih inventaris bernilai puluhan juta rupiah.

The Architecture (Desain Sistem)

Ekosistem S2SMFG dibangun di atas arsitektur monolitik Laravel dengan pendekatan modular. Database MariaDB menampung 78 tabel relasional yang dinormalisasi hingga bentuk 3NF untuk menjaga integritas data transaksi tinggi.

1. Raw Materials
Pencatatan pengadaan & bahan baku masuk gudang.
→
2. Barcode SPK
Sistem otomatisasi cetak barcode SPK & Label Box.
→
3. QC Scanner
Validasi QR Code barang keluar & QC checking.
→
4. Finish Good Out
Logistik armada & sinkronisasi data surat jalan.

Setiap box barang jadi yang diproduksi diberi label QR Code unik berisi data nomor SPK, ID part, jumlah isi box, dan timestamp. Operator QC di stasiun akhir cukup memindai QR Code tersebut untuk merekam status "Good" secara real-time.

The Code (Implementasi Transaksi)

Untuk memastikan tidak terjadi race condition atau data corrupt saat 50+ operator secara bersamaan melakukan scan QR Code barang keluar, seluruh operasi tulis database dibungkus di dalam Database Transaction Laravel.

Berikut adalah cuplikan kode backend di controller yang menangani perekaman scan barang keluar dengan mekanisme validasi ketat: 

// App/Http/Controllers/Api/PrintQueueController.php
namespace App\Http\Controllers\Api;

use Illuminate\Http\Request;
use Illuminate\Support\Facades\DB;
use App\Models\TFinishGoodOut;
use App\Models\shipping\TSpk;

class FinishGoodScanController extends Controller
{
    /**
     * Memproses scan QR code box finished good untuk pengiriman.
     * Menggunakan DB transaction untuk menjamin konsistensi data atomik.
     */
    public function processScan(Request $request)
    {
        $request->validate([
            'spk_id' => 'required|exists:t_spk,id',
            'qr_content' => 'required|string',
            'qty' => 'required|integer|min:1'
        ]);

        // Parsing konten QR (Format: PART-CODE|CYCLE|BOX-NUM)
        $parsedData = explode('|', $request->qr_content);
        if (count($parsedData) < 3) {
            return response()->json(['status' => 'error', 'message' => 'Format QR tidak valid'], 400);
        }

        $partCode = $parsedData[0];
        $cycle = $parsedData[1];

        // Jalankan Database Transaction untuk menjamin atomic save & lock
        return DB::transaction(function() use ($request, $partCode, $cycle) {
            // 1. Lock Row SPK untuk update balance pengiriman
            $spk = TSpk::where('id', $request->spk_id)->lockForUpdate()->firstOrFail();

            // 2. Periksa duplikasi scan box unik
            $exists = TFinishGoodOut::where('qr_hash', md5($request->qr_content))
                ->where('status', 'scanned')
                ->exists();

            if ($exists) {
                return response()->json(['status' => 'error', 'message' => 'Box ini sudah discan sebelumnya'], 422);
            }

            // 3. Catat scan finished good
            $fgScan = new TFinishGoodOut();
            $fgScan->spk_id = $spk->id;
            $fgScan->part_code = $partCode;
            $fgScan->cycle = $cycle;
            $fgScan->qty = $request->qty;
            $fgScan->qr_hash = md5($request->qr_content);
            $fgScan->status = 'scanned';
            $fgScan->save();

            // 4. Update status tracking SPK
            $spk->increment('total_scanned_qty', $request->qty);

            return response()->json([
                'status' => 'success',
                'message' => 'Box berhasil divalidasi',
                'data' => [
                    'part_code' => $partCode,
                    'qty' => $request->qty
                ]
            ]);
        });
    }
}

Mengapa kode ini scalable?

  • Menggunakan DB::transaction untuk memastikan konsistensi relasi. Jika pencatatan scan berhasil tetapi update SPK gagal, seluruh data akan otomatis di-rollback.
  • Fungsi lockForUpdate() mengunci record baris SPK pada database saat transaksi berjalan, memblokir request lain dari mengubah SPK yang sama secara bersamaan guna mencegah selisih stok (oversell/over-ship).
  • Validasi keunikan berbasis hash MD5 mencegah manipulasi data atau double scan box oleh operator.
