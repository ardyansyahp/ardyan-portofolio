03. Case Study: Analytics & Control

Asakai Dashboard & DataQC

Klien: PT. Mada Wikri Tunggal
Peran: Lead Developer
Stack: Laravel 12, Livewire 3, Alpine.js, Tailwind CSS
Optimasi: Agregasi Otomatis TV Monitor

The Problem (Masalah Bisnis)

Setiap pagi jajaran manajemen mengadakan rapat koordinasi cepat (Morning Briefing / Asakai) untuk mengevaluasi parameter operasional dari hari sebelumnya. Sebelum dashboard Asakai dideploy:

  • Persiapan Laporan Manual: Setiap kepala departemen (Produksi, PPIC, QC, Moldshop, Maintenance, HSE, & GA) harus meluangkan waktu 1–2 jam per hari untuk mengumpulkan data dari lembaran kertas dan mencatatnya ke file Excel.
  • Keterlambatan Informasi: Data sering kali baru selesai direkap siang hari, padahal rapat dilakukan jam 8 pagi. Rapat sering kali berjalan menggunakan data yang sudah usang atau estimasi verbal yang tidak akurat.
  • Kesulitan Visualisasi: Laporan disajikan dalam bentuk spreadsheet rumit yang sulit dibaca dari jarak jauh pada layar monitor TV ruang rapat.

The Architecture (Command Center Manufaktur)

Asakai Dashboard dirancang sebagai single source of truth yang menyatukan data operasional dari 7 departemen berbeda. Sistem ini diletakkan pada TV besar di dinding pabrik dan ruang manajemen, teroptimasi untuk konsumsi visual kepadatan data tinggi (High Data Density).

7 Dept Databases
Model HSE, PPIC, Produksi, QC, Moldshop, Maintenance, GA.
→
Laravel Controller
Optimasi query database agregasi (sum, count, average) terintegrasi Carbon.
→
Livewire Reactivity
Server-side polling & refresh data otomatis tanpa memuat ulang (reload) halaman browser.
→
Command Center TV
Visualisasi grid minimalis bertema gelap, dinavigasi pintasan keyboard (Arrow Keys).

Sistem ini mengandalkan integrasi Tailwind CSS, Alpine.js, Laravel, dan Livewire untuk memberikan pembaruan antarmuka secara instan dari server tanpa perlu menulis file JavaScript SPA yang berat dan berlebihan.

The Code (Optimasi Aggregasi Multi-Tabel)

Untuk menyajikan data dari 7 departemen dalam sekali waktu tanpa membebani memori server, query Eloquent Laravel dioptimalkan untuk melakukan kalkulasi agregasi langsung di tingkat database (MySQL/MariaDB) daripada memuat record ke memori PHP.

Berikut adalah cuplikan controller backend yang menyusun agregasi metrik dari departemen HSE, PPIC, QC, hingga Moldshop: 

// App/Http/Controllers/DashboardController.php
namespace App\Http\Controllers;

use Carbon\Carbon;
use App\Models\HseAccident;
use App\Models\BiraProblem;
use App\Models\Kaizen;
use App\Models\ProduksiOee;
use App\Models\QcClaimPpm;
use App\Models\MoldshopPreventive;

class DashboardController extends Controller
{
    public function index()
    {
        $now = Carbon::now();
        $currentYear = $now->year;

        // --- HSE (Safety, Inovasi, & Kaizen) ---
        // Menggunakan sum() dan count() secara langsung di database
        $hseAccidentTotal = HseAccident::whereYear('date', $currentYear)->sum('frequency') ?? 0;
        $biraOpen         = BiraProblem::where('status', 'Open')->count();
        $kaizenTotal      = Kaizen::whereYear('created_at', $currentYear)->count();

        // --- PRODUKSI ---
        // Mendapatkan nilai Overall Equipment Effectiveness (OEE) terbaru harian
        $produksiLatestOee = ProduksiOee::orderBy('date', 'desc')->first();

        // --- QC ---
        // Klaim kualitas pelanggan (Customer Claim PPM)
        $qcLatestClaim = QcClaimPpm::orderBy('date', 'desc')->first();

        // --- MOLDSHOP ---
        // Jumlah perawatan pencegahan mesin (Preventive Maintenance) tahun berjalan
        $moldshopScheduleTotal = MoldshopPreventive::whereYear('actual_date', $currentYear)
            ->whereNotNull('actual_date')
            ->count();

        // Struktur departemen untuk pemetaan dinamis
        $departments = [
            ['name' => 'HSE', 'records' => HseAccident::count() + BiraProblem::count() + Kaizen::count()],
            ['name' => 'Produksi', 'records' => ProduksiOee::count()],
            ['name' => 'QC', 'records' => QcClaimPpm::count()],
            ['name' => 'Moldshop', 'records' => MoldshopPreventive::count()]
        ];

        $totalRecords = collect($departments)->sum('records');

        return view('welcome', compact(
            'hseAccidentTotal',
            'biraOpen',
            'kaizenTotal',
            'produksiLatestOee',
            'qcLatestClaim',
            'moldshopScheduleTotal',
            'totalRecords',
            'departments'
        ]));
    }
}

Mengapa arsitektur ini berdampak tinggi?

  • Penyajian Instan: Begitu jam kerja rapat pagi dimulai, manajemen cukup menekan pintasan tombol untuk menampilkan halaman TV monitor Asakai yang telah menarik seluruh ringkasan KPI secara real-time. Persiapan manual dipangkas habis.
  • Peningkatan Akurasi: Karena data ditarik dari basis data produksi S2SMFG secara otomatis (misalnya data QC claim, pencatatan downtime mesin), potensi bias atau manipulasi data laporan telah dieliminasi.
  • Navigasi Fleksibel: Presenter dapat menelusuri data antar departemen dengan pintasan keyboard terintegrasi Alpine.js untuk meminimalkan ketergantungan pada mouse saat melakukan pemaparan rapat.
