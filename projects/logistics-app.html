04. Case Study: Mobile App & Integration

Logistics Driver App

Klien: PT. Mada Wikri Tunggal
Peran: Mobile & Backend Developer
Stack: Flutter (Dart), GetX, GPS, QR, Laravel API
Akurasi: 95% Konfirmasi Lebih Cepat

The Problem (Masalah Bisnis)

Sebelum aplikasi seluler logistik dideploy, manajemen tidak memiliki visibilitas terhadap status pengiriman barang dari pabrik ke stasiun perakitan pelanggan (customer plant):

  • Buta Lokasi Armada: Tim logistik tidak mengetahui posisi truk pengiriman secara real-time, sehingga harus berulang kali menelepon pengemudi saat di jalan.
  • Proses Manual Lambat: Konfirmasi penerimaan barang hanya bisa diverifikasi setelah pengemudi kembali ke pabrik dan menyerahkan lembar fisik Surat Jalan bertanda tangan.
  • Sengketa Waktu Pengiriman: Ketiadaan timestamp kedatangan digital yang presisi menyulitkan verifikasi klaim keterlambatan pasokan (just-in-time penalty) dari pelanggan.

The Architecture (Sinkronisasi GPS & QR)

Solusinya adalah aplikasi mobile cross-platform berbasis Flutter untuk platform Android yang dipasang pada handphone seluruh driver armada pabrik. Aplikasi ini terintegrasi langsung dengan API backend Laravel di cloud.

1. Flutter Driver Client
Memuat daftar trip surat jalan. Mengakses modul GPS & Scanner QR hardware HP.
→
2. Geo-Location Loop
Perekaman koordinat lokasi (Latitude/Longitude) secara periodik saat perjalanan aktif.
→
3. Scan QR on Arrival
Driver memindai QR Code di gerbang pelanggan untuk memicu tanda terima digital.
→
4. Laravel API & DB
Menyimpan timestamp log presisi GPS untuk diverifikasi oleh tim internal PPIC.

Driver menggunakan pemindai QR bawaan aplikasi untuk men-scan kode di plant tujuan. Jika koordinat GPS driver berada di luar radius radius pagar (geofencing) plant pelanggan, sistem akan menolak konfirmasi masuk untuk mencegah kecurangan pengisian laporan kedatangan.

The Code (GetX State & Endpoint Sinkronisasi)

Aplikasi mobile menggunakan GetX untuk manajemen state dan dependensi. Berikut adalah cuplikan controller Flutter (Dart) yang menangani sinkronisasi lokasi GPS dan payload konfirmasi pengiriman: 

// driver_app/lib/app/controllers/delivery_controller.dart
import 'package:get/get.dart';
import 'package:geolocator/geolocator.dart';
import 'package:http/http.dart' as http;
import 'dart:convert';

class DeliveryController extends GetxController {
  var isLoading = false.obs;
  var currentPosition = Rxn<Position>();

  /// Memicu konfirmasi kedatangan di lokasi pengiriman
  Future<bool> confirmArrival(int spkId, String qrCodeContent) async {
    try {
      isLoading.value = true;
      
      // 1. Ambil koordinat GPS terkini dari perangkat
      Position position = await Geolocator.getCurrentPosition(
        desiredAccuracy: LocationAccuracy.high
      );
      currentPosition.value = position;

      // 2. Kirim payload data ke REST API Laravel ERP
      final response = await http.post(
        Uri.parse('https://s2smfg.biz.id/api/delivery/arrival-confirm'),
        headers: {'Content-Type': 'application/json'},
        body: jsonEncode({
          'spk_id': spkId,
          'qr_content': qrCodeContent,
          'latitude': position.latitude,
          'longitude': position.longitude,
          'scanned_at': DateTime.now().toIso8601String()
        })
      );

      if (response.statusCode == 200) {
        Get.snackbar('Sukses', 'Konfirmasi kedatangan berhasil dikirim');
        return true;
      } else {
        final errData = jsonDecode(response.body);
        Get.defaultDialog(title: 'Gagal', middleText: errData['message']);
        return false;
      }
    } catch (e) {
      Get.snackbar('Error Connection', e.toString());
      return false;
    } finally {
      isLoading.value = false;
    }
  }
}

Berikut adalah cuplikan controller API di backend Laravel yang memproses data kedatangan driver dan menghitung deviasi jarak: 

// App/Http/Controllers/Api/DeliveryApiController.php
public function confirmArrival(Request $request)
{
    $request->validate([
        'spk_id' => 'required|exists:t_spk,id',
        'latitude' => 'required|numeric',
        'longitude' => 'required|numeric'
    ]);

    $spk = TSpk::with('customer')->findOrFail($request->spk_id);
    
    // Hitung jarak (Formula Haversine) antara GPS driver & koordinat plant pelanggan
    $latFrom = deg2rad($request->latitude);
    $lonFrom = deg2rad($request->longitude);
    $latTo   = deg2rad($spk->customer->latitude);
    $lonTo   = deg2rad($spk->customer->longitude);

    $earthRadius = 6371000; // Meter
    $latDelta = $latTo - $latFrom;
    $lonDelta = $lonTo - $lonFrom;

    $angle = 2 * asin(sqrt(pow(sin($latDelta / 2), 2) +
      cos($latFrom) * cos($latTo) * pow(sin($lonDelta / 2), 2));
    
    $distance = $angle * $earthRadius; // Jarak deviasi

    // Toleransi geofence maksimal 200 meter dari gerbang customer
    if ($distance > 200) {
        return response()->json([
            'status' => 'error',
            'message' => 'Posisi Anda terlalu jauh dari lokasi customer (' . round($distance) . ' meter)'
        ], 400);
    }

    // Catat status kedatangan ke basis data
    $spk->update([
        'status' => 'arrived',
        'actual_arrival_time' => now(),
        'arrival_latitude' => $request->latitude,
        'arrival_longitude' => $request->longitude
    ]);

    return response()->json(['status' => 'success', 'message' => 'Kedatangan terverifikasi']);
}

Mengapa integrasi ini tangguh?

  • Proteksi Geofencing: Perhitungan rumus Haversine langsung di sisi backend mencegah manipulasi spoofing GPS palsu oleh driver demi mendapatkan validasi awal palsu.
  • GetX Reactivity: Menggunakan observables GetX (.obs) untuk memperbarui state UI pemindaian secara real-time tanpa penundaan rendering layar HP driver.
  • Data Terintegrasi: Logistik terhubung langsung ke ERP, memangkas siklus pelacakan surat jalan dari 1 hari menjadi kurang dari 1 detik sejak driver men-scan QR kedatangan di customer plant.
