04. Case Study: Mobile App & Integration

Logistics Driver App

Klien: PT. Mada Wikri Tunggal
Peran: Lead Mobile Developer (Flutter)
Tingkat Adopsi: 100% (Oleh 20 Driver Operasional)
Stack: Flutter, GetX, GPS, QR Scanner, REST API

Masalah Operasional Awal

Pelacakan armada pengiriman barang jadi (finished goods) ke customer plant sebelumnya dilakukan secara manual. Gudang PPIC tidak memiliki visibilitas langsung terhadap status pengiriman barang dari pabrik ke stasiun perakitan pelanggan:

  • Buta Lokasi Armada: Tim logistik tidak mengetahui posisi truk pengiriman secara real-time, sehingga harus berulang kali menelpon pengemudi saat di jalan.
  • Proses Manual Lambat: Konfirmasi penerimaan barang hanya bisa divalidasi setelah pengemudi kembali ke pabrik pada sore hari untuk menyerahkan fisik Surat Jalan bertanda tangan.
  • Sengketa Waktu Pengiriman: Ketiadaan catatan timestamp digital yang akurat menyulitkan penelusuran jika terjadi klaim keterlambatan pasokan (just-in-time) oleh pelanggan.

Pendekatan Arsitektur Teknologi yang Pilih

Solusinya adalah aplikasi mobile cross-platform berbasis Flutter untuk platform Android yang dipasang pada handphone seluruh driver armada pabrik. Aplikasi ini terintegrasi langsung dengan API backend Laravel di cloud.

1. Flutter Driver Client
Memuat daftar trip surat jalan. Mengakses modul GPS & Scanner QR HP.
→
2. Geo-Location Loop
Perekaman koordinat lokasi (Latitude/Longitude) secara periodik saat perjalanan aktif.
→
3. Scan QR on Arrival
Driver memindai QR Code di gerbang pelanggan untuk memicu tanda terima digital.
→
4. Laravel API & DB
Menyimpan timestamp log presisi GPS untuk diverifikasi oleh tim internal PPIC.

Driver menggunakan pemindai QR bawaan aplikasi untuk men-scan kode di plant tujuan. Jika koordinat GPS driver berada di luar radius radius pagar (geofencing) plant pelanggan, sistem akan menolak konfirmasi masuk untuk mencegah kecurangan pengisian laporan kedatangan.

Tantangan Teknis & Integrasi Flutter-REST API

Tantangan teknis terbesar adalah merancang sinkronisasi geofencing yang hemat baterai serta mencegah kecurangan (GPS Spoofing) oleh driver di lapangan.

Solusi GetX & Haversine Geofencing:
State management diatur menggunakan GetX untuk memproses perubahan status pengiriman secara responsif. Rumus Haversine dihitung langsung di sisi backend Laravel saat API menerima payload koordinat, memastikan validasi jarak fisik driver nyata dari pagar koordinat plant pelanggan sebelum menyetujui transaksi kedatangan.

Berikut adalah cuplikan controller Flutter (Dart) yang menangani pemindaian dan pengiriman koordinat: 

// driver_app/lib/app/controllers/delivery_controller.dart
import 'package:get/get.dart';
import 'package:geolocator/geolocator.dart';
import 'package:http/http.dart' as http;
import 'dart:convert';

class DeliveryController extends GetxController {
  var isLoading = false.obs;
  var currentPosition = Rxn<Position>();

  Future<bool> confirmArrival(int spkId, String qrCodeContent) async {
    try {
      isLoading.value = true;
      
      // Ambil koordinat GPS terkini perangkat driver
      Position position = await Geolocator.getCurrentPosition(
        desiredAccuracy: LocationAccuracy.high
      );
      currentPosition.value = position;

      // Kirim ke REST API server Laravel
      final response = await http.post(
        Uri.parse('https://s2smfg.biz.id/api/delivery/arrival-confirm'),
        headers: {'Content-Type': 'application/json'},
        body: jsonEncode({
          'spk_id': spkId,
          'qr_content': qrCodeContent,
          'latitude': position.latitude,
          'longitude': position.longitude
        })
      );

      return response.statusCode == 200;
    } catch (e) {
      return false;
    } finally {
      isLoading.value = false;
    }
  }
}

Hasil Akhir yang Terukur

Aplikasi mobile Android ini sukses disebarkan secara instan ke seluruh armada pengiriman. Efisiensi pencatatan waktu kirim meningkat pesat secara kuantitatif:

[ FORMULA EFISIENSI WAKTU PELAPORAN ]
ΔTpelaporan = (Tmanual - Taplikasi) / Tmanual × 100%
ΔTpelaporan = (28800 s - 10 s) / 28800 s × 100% ≈ 99.9% (Real-time Sync)

[ TINGKAT ADOPSI ARMADA ]
Adopsi Driver = 20 Driver Aktif / 20 Driver Armada = 100%

Dampak Terukur Hasil Penerapan:

  • Kecepatan Rekap Laporan: Dari rekap manual lembar fisik ke Excel di sore hari (delay 8 jam) menjadi instan 10 detik saat QR discan di plant customer.
  • Akurasi Pelacakan GPS Trip: Mengeliminasi komplain klaim keterlambatan sepihak dari pelanggan karena sistem mencatat koordinat dan waktu tiba presisi.
  • Tingkat Adopsi Driver: 100% adopsi sukses dari 20 driver berkat desain UI/UX tombol yang ramah bagi operator.
